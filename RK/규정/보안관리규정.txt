1. 조항 번호 오류
    👉 문서 신뢰성 문제

    기존: 제8조가 여러 번 반복됨 (시설 / 방문자 등)
    문제:
        → 공식 규정 문서에서 조항 충돌 = 문서 무효/혼란 가능
    개선:
        → 조항 번호를 순차적으로 재정렬 (제15조, 제16조 등)

    📌 핵심
        → “형식 문제지만 감사/법무에서 바로 지적되는 부분”

2. IT 보안 부족
    👉 실제 리스크 방치 상태

    - 기존: “컴퓨터 보안” 수준만 존재 (전원 끄기 등)
    - 빠진 요소:
        > 계정 관리 (개인계정 사용)
        > MFA (다중인증)
        > 클라우드 (M365 / OneDrive)
        > USB / 외부 저장장치
        > 프로그램 설치 통제
    - 개선:
        → “계정 / 인증 / 클라우드 / 전송” 기준 추가

    📌 핵심
    → 지금 시대 기준에서 가장 중요한 보안 영역이 빠져 있었음

3. 자산관리 없음

    👉 RK 환경에서 가장 큰 리스크

    - 기존: 장비에 대한 규정 거의 없음
    - 문제 상황:
        > 노트북 지급 / 회수
        > 휴대폰 교체
        > 중고 매각
        > 외부 반출
    - 개선:
        → 자산 등록 / 반출 승인 / 폐기 시 데이터 삭제 명시

    📌 핵심
    → “보안 = 장비 관리” 수준이라 반드시 필요

4. 퇴직자 보안 약함

    👉 실제 사고 대부분이 여기서 발생

    - 기존:
        → 단순 “퇴직 후 비밀유지”
    - 부족한 부분:
        > 계정 즉시 차단
        > 시스템 접근 제거
        > 장비 회수
        > 데이터 삭제
    - 개선:
    → 퇴직 시 접근 차단 + 자산 회수 + 데이터 보호 명확화

    📌 핵심
    → 내부 유출은 대부분 “퇴직 시점”에서 발생

5. 사고 대응 없음

    👉 사고 발생 시 대응 불가능 상태

    - 기존:
        → 사고 발생 시 규정 없음
    - 문제:
        > 누가 보고?
        > 누가 대응?
        > 어떻게 처리?
    - 개선:
        → “즉시 보고 → 보안책임자 대응 → 원인 분석” 구조 추가

    📌 핵심
    → 규정이 없으면 사고 나도 책임/절차 없음

6. 법적 보호 부족

    👉 회사 보호 장치 부족

    - 기존:
        → “비밀유지” 정도만 있음
    - 부족:
        > 영업비밀 정의 없음
        > 위반 시 책임 불명확
        > 징계 / 법적 조치 없음
    - 개선:
        → “징계 가능 + 민형사 책임 가능” 명시

    📌 핵심
    → 분쟁 발생 시 회사 보호 근거가 없음 → 반드시 필요
