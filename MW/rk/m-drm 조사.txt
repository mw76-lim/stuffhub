-- DRM 조사
2️. RK에서 DRM 리서치가 필요한 자산 범위 (우선순위 기준)
    🔴 Level 1. 반드시 커버해야 할 핵심 영역 (필수)
        ① 기술 문서 & 설계 자산
        - 공정 설명서 (Process Flow, Spec)
        - 제품 데이터시트 (Si / Quartz / SiC / Gasket 등)
        - 고객사 제출용 기술 제안서
        - 장비 매뉴얼 / 설치 가이드
        - 도면 (PDF, CAD, 이미지)
        📌 DRM 필요 포인트
        - 열람만 가능 / 다운로드 금지
        - 캡처 방지
        - 인쇄 제한
        - 고객사·협력사별 접근 권한 분리
        - 열람 로그 추적
        ② 내부 공유 문서 (Microsoft 365 / OneDrive / WEHAGO 연계)
        - 사내 교육자료
        - 기술 세미나 자료
        - 회의 자료
        - 내부 가이드 문서
        📌 DRM 필요 포인트
        - 퇴사자 접근 자동 차단
        - 외부 공유 시 만료 기간 설정
        - 재배포 방지
        - 회사 계정 외 열람 제한

    🟠 Level 2. 전략적으로 중요하지만 선택 적용 영역
        ③ 고객·파트너 대상 자료
        - NDA 하에 제공되는 자료
        - IR / PR 사전 공유 자료
        - 공동 개발 관련 기술 문서
        📌 DRM 필요 포인트
        - 회사·프로젝트 단위 권한
        - NDA 종료 시 접근 자동 회수
        - 열람 횟수/기간 제한
        ④ 영상 자료 (교육 / 세미나 / 장비 설명)
        - 사내 교육 영상
        - 고객용 기술 설명 영상
        - 장비 설치·운영 영상
        📌 DRM 필요 포인트
        - 스트리밍 전용 (다운로드 불가)
        - 워터마크 삽입
        - 계정 기반 시청

    🟡 Level 3. RK에서는 낮은 우선순위 (리서치만)
        ⑤ 상업용 콘텐츠 DRM (OTT/음원 등)
        - 영화, 음악, 일반 엔터테인먼트 콘텐츠 보호

        ➡️ RK에는 직접 적용성 낮음
        → 구조 이해 정도만 리서치하면 충분

3️. DRM 기술 유형별 분류 (리서치 프레임)
    A. 문서 중심 DRM (RK 핵심)
        - PDF / Office / 이미지 파일 제어
        - 권한 기반 접근
        - 로그 추적
        대표 기술/플랫폼:
        - Microsoft Purview
        - Azure Information Protection
        - Seclore
        - Fasoo
    B. 클라우드·계정 기반 DRM
        - 회사 계정에 종속된 접근 제어
        - 파일이 외부로 나가도 권한 유지
        📌 RK의 Microsoft 365 + OneDrive + WEHAGO 구조와 궁합이 좋음
    C. 워터마킹 & 추적 DRM
        - 문서·영상에 사용자 정보 삽입
        - 유출 시 책임 추적 가능
    D. 영상 DRM (보조 영역)
        - 스트리밍 보호
        - 다운로드 차단
        대표 기술:
        - Google Widevine
        - Apple FairPlay

4️. RK 리서치 문서 구조 (이대로 정리하면 좋음)
    ① DRM 도입 배경 (RK 관점)
    - 기술 유출 리스크
    - 글로벌 고객 대응
    - 협력사·외주 증가
    ② 보호 대상 자산 정의
    - 기술 / 영업 / 교육 / 영상
    ③ DRM 적용 시나리오
    - 사내
    - 고객사 제공
    - 협력사 제공
    - 퇴사/프로젝트 종료
    ④ 기술 유형별 비교
    - 문서 DRM vs 계정 기반
    - 서버형 vs 클라우드형
    ⑤ RK 환경과의 적합성
    - Microsoft 365 연동성
    - 운영 부담
    - 비용 구조
    
5️. 한 줄로 정리하면
    RK에서 DRM 리서치는
    “기술 문서와 내부 지식 자산을
    회사 계정과 권한 중심으로 통제하는 체계”를
    기준으로 보면 충분하다.
