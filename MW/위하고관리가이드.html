론다코리아 WEHAGO IT 운영 가이드

IT 관리자 / 내부 기준용 · 1 Page

ADMIN · INTERNAL
1

문서 목적

본 문서는 론다코리아의 WEHAGO 사용 환경을 기업 IT 인프라 관점에서 안정적·보안적으로 운영하기 위한 관리자 기준 가이드입니다.

핵심 원칙 WEHAGO는 단순 업무툴이 아니라 계정·권한·데이터가 결합된 기업 IT 자산으로 관리합니다.
2

서비스 성격 정의

  • WEHAGO는 SaaS(클라우드 기반) 서비스
  • 서버·DB·업데이트는 WEHAGO 제공사 책임
  • 계정·권한·보안·연동·데이터 정책은 회사 IT 책임
구분 “서버 관리 ❌ / 운영·통제 관리 ⭕”
3

IT 관리자가 반드시 관장해야 할 영역

① 계정 관리 (가장 중요)

  • 임직원 계정 생성 / 수정 / 삭제
  • 퇴사자 계정 즉시 차단
  • 외주·협력사 계정 기간·권한 제한
  • 개인 이메일 계정 사용 ❌
기준 모든 계정은 회사 자산이며, 개인 소유가 아닙니다.

② 권한 관리 (Role & Access)

  • 부서·직무별 권한 분리
  • 회계 / 인사 / 급여 / IR 문서 접근 최소화
  • “필요한 사람만, 필요한 만큼” 원칙 적용
예시 회계팀 → 재무/급여 접근 · 마케팅 → 디자인/콘텐츠 접근 · 외주 → 읽기 전용 또는 폴더 한정

③ 보안 정책 설정 (Security Configuration)

IT 관리자는 아래 항목을 반드시 설정·점검해야 합니다.

  • 2단계 인증(MFA) 사용 여부
  • 외부 접속 허용 범위
  • 외부 공유(링크) 제한
  • 비밀번호 정책
원칙 편의보다 보안을 우선합니다.

④ 데이터 관리 & 외부 공유

  • 문서 외부 공유는 최소화
  • 외부 공유 시: 기간 제한 / 다운로드 제한 / 대상 명확화
  • 장기 보관 문서는 공용 공간 기준 관리
금지 개인 계정으로 회사 자료 이전 · 무기한 외부 공유 링크

⑤ 시스템 연동 관리

WEHAGO는 이메일, ERP/회계, 전자결재, Microsoft 365 등 외부 업무도구와 연동될 수 있습니다.

  • 연동 범위 승인
  • 데이터 흐름 확인
  • 장애 발생 시 책임 주체 명확화

⑥ 로그·이력·감사 대응

  • 로그인 기록
  • 권한 변경 이력
  • 문서 접근 로그
목적 내부 감사 · 보안 사고 대응 · 외부 감사 대응
4

IT 관리 대상이 아닌 영역

항목 관리 주체
서버 운영 WEHAGO
DB 백업 WEHAGO
시스템 업데이트 WEHAGO
서비스 가용성 WEHAGO
정리 IT 인력은 “서버 관리자”가 아니라 운영·통제 관리자입니다.
5

퇴사·외주 인력 관리 기준

퇴사자

  1. 계정 즉시 비활성화
  2. 자료 이관 여부 확인
  3. 접근 로그 점검

외주/협력사

  • 전용 계정 발급
  • 접근 범위 제한
  • 종료일 명확화
6

한 줄 요약 (임원 보고용)

요약 WEHAGO는 서버를 관리하지 않아도 되지만, 계정·권한·보안·데이터 흐름은 반드시 IT 인력이 통제해야 하는 기업 IT 인프라입니다.
7

권장 후속 문서

  • 임직원용 WEHAGO 사용 가이드 (행동 기준)
  • 외주 인력 계정 발급·회수 프로세스
  • Microsoft 365 / OneDrive 연계 책임 구분표
